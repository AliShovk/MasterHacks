# Инструкция для покупателя

## Что это за продукт

Это готовая медиа-платформа с загрузкой контента через Telegram-бота.

В состав системы входят:

- загрузка фото и видео через Telegram
- PHP backend
- MySQL / MariaDB
- модерация контента
- публичная лента контента
- webhook-интеграция с Telegram Bot API
- конфигурация через `.env`

## Что должно передаваться покупателю

- исходный код проекта
- файл `.env.example`
- SQL-дамп базы данных
- эта инструкция
- чеклист передачи
- карточка проекта для продажи

## Что нужно покупателю до запуска

- домен
- VPS или хостинг с PHP
- Nginx или Apache
- PHP 8.1+
- MySQL или MariaDB
- SSL сертификат
- Telegram-бот, созданный через BotFather

## Быстрый запуск

### 1. Загрузить проект на сервер

Разместить проект в директории сайта, например:

- `/var/www/project`

### 2. Создать файл `.env`

Скопировать `.env.example` в `.env` и заполнить своими данными.

Пример:

```env
PROJECT_NAME=MyMediaProject
USE_DB=true
SITE_URL=https://example.com
ADMIN_PANEL_KEY=change_me_admin_key
ADMIN_CHAT_ID=

TELEGRAM_BOT_TOKEN=123456:ABCDEF
TELEGRAM_BOT_USERNAME=my_upload_bot
TELEGRAM_BOT_URL=https://t.me/my_upload_bot

DB_HOST=localhost
DB_PORT=3306
DB_NAME=myproject_db
DB_USER=myproject_user
DB_PASS=myproject_password
```

### 3. Создать базу данных

Создать БД и пользователя, затем импортировать SQL.

Можно использовать:

- полный дамп: `sql/masterhacks_fixed.sql`
- только структуру: `sql/schema.sql`

### 4. Настроить веб-сервер

Требования:

- сайт должен открываться из корня проекта
- PHP должен быть подключён
- HTTPS должен работать корректно
- у проекта должны быть права записи в `media/` и используемые папки данных/кэша

### 5. Установить webhook Telegram

После деплоя открыть в браузере:

- `/setWebhook.php`

или:

- `/set_webhook.php`

Ожидаемый результат:

- webhook успешно установлен
- Telegram не возвращает SSL-ошибку
- токен бота читается корректно

### 6. Проверить работу бота

- открыть бота покупателя в Telegram
- отправить `/start`
- отправить фото или видео
- убедиться, что контент сохраняется
- убедиться, что контент попадает в модерацию или ленту по текущей логике проекта

## Основные переменные `.env`

### Обязательные

- `SITE_URL`
- `TELEGRAM_BOT_TOKEN`
- `TELEGRAM_BOT_USERNAME`
- `TELEGRAM_BOT_URL`
- `DB_HOST`
- `DB_PORT`
- `DB_NAME`
- `DB_USER`
- `DB_PASS`
- `ADMIN_PANEL_KEY`

### Дополнительные

- `PROJECT_NAME`
- `ADMIN_CHAT_ID`

## Что обязательно должно быть уникальным у каждого покупателя

- домен
- SSL сертификат
- Telegram bot token
- username Telegram-бота
- база данных
- логин и пароль БД
- admin key
- брендовые данные

## Чек после установки

- сайт открывается по HTTPS
- webhook активен
- бот принимает загрузки
- база данных подключается
- файлы сохраняются
- модерация работает
- лента открывается без ошибок

## Рекомендации по безопасности

- не хранить рабочий `.env` в публичном репозитории
- не использовать один Telegram-бот для разных клиентов
- не использовать одну общую базу данных для всех покупателей
- передавать покупателю новый `ADMIN_PANEL_KEY`
- ограничить доступ к серверу и панели администратора

## Важно для white-label продажи

Технические настройки уже вынесены в `.env`, но для полной перепродажи под другим брендом покупателю может понадобиться отдельно заменить:

- название проекта
- логотип
- favicon
- отдельные тексты интерфейса
- контактные данные
